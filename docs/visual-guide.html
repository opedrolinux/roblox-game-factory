The Game Factory 🏭

You describe a game in one page. Claude Code builds it for you — for hours, on its own, checking its own work — and only taps you on the shoulder for the few decisions a human has to make. And because it's a factory, the same machine makes game after game.

1 · The big picture

Like a real factory: an idea goes in one door, a finished game comes out the other. Inside there's a reusable blueprint, an assembly line, worker robots, and quality control. The "robots" are Claude agents; the "product" is a Roblox game. You give it a look and publish at the end.

2 · How a game gets made

  1. Spec — your one-page description.
  2. Copy the blueprint — start from the shared foundation, never from zero.
  3. Lay the shared wiring once — so the robots don't trip over each other.
  4. Split into features — each robot builds its piece and writes tests for it.
  5. 🧪 TEST GATE per feature — a separate testing robot writes fresh tests from your spec; the feature can't merge until it's green.
  6. Merge everything — keep every robot's work.
  7. 🧪 TEST GATE after merge — test the whole combined game; nothing that worked before is broken.
  8. Hunt bugs & exploits — robots try to break it, then fix it.
  9. Full test pass — the whole game must go green.
  10. Your turn — check the look & feel, then publish.
Why test at every step? The robot that built a feature is biased toward its own work. A separate tester, writing tests from your spec instead of from the code, catches the bugs the builder can't see — so problems are caught at the smallest scope, before they pile up.

3 · Many robots, no collisions

Each robot gets its own private copy of the project (a "worktree"), so they can't overwrite each other. When done, their work is carefully merged and re-tested. This is why a Max-20x subscription matters: you're paying a flat price to run many robots at once, for hours — idle time is wasted money.

4 · What it does alone — and where it stops for you

GREEN just does it: writing code, running tests, saving progress — safe, reversible stuff, no nagging.

YELLOW waits for you: does it look good? should we publish? grow or kill it?

RED never alone: publishing, spending money, deleting/undoing, touching your account — locked at the tool level.

Safety net: a checkpoint after every step, plus OneDrive backup. Nothing is truly lost.

5 · Why we run it this way (the money part)

From June 15, 2026, keeping a normal session open and busy is already paid for (flat rate), but fully unattended background runs are metered. So we keep the thinking in the free lane and only use pay-per-use for tiny chores (like the final upload). You can run it hard for hours and stay on the flat price.
